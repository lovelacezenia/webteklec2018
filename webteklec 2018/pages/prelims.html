    Hyper Text Transfer Protocol (HTTP) Fundamentals
  • HTTP typically runs on top of TCP/IP, using TCP port 80 by default (TCP port 443 for HTTPS).
  • HTTP resources are identified using URIs (specifically HTTP URLs).
  • HTTP is based on client-server architecture.
  • HTTP is not the most visible but the corner stone of how web works.
  • HTTP is an application layer communication protocol used to access resources (hypertext/hypermedia)on WWW.
  • HTTP is jointly developed by the W3C and the IETF protocol
  • HTTP is a pull protocol, it is always the client who initiates or pull communication.
    • - Scheme (http: or https:)
    • - (optional) authentication information
    • - Host and (optional) port number
    • - Path (resolved to the document root on the server) to the resource
    • - (optional) scheme-specific parameters
    • - (optional) URL-encoded query
    • - (optional) bookmark (or fragment identifier)
  • Clients (AKA User Agents (UA)).
    • - Web browsers, Web crawlers, Email clients, Other end user tools and applications
  • Servers
    • - Origin servers, Proxy servers, Gateways, Tunnels
  • HTTP uses a request-response standard protocol.
    • - Client sends an HTTP request to the server.
    • - Server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol.
    • - Servers DO NOT keep information about clients between requests.
    • - Web applications affect session tracking using mechanisms such as cookies (a very small text file) on URL-encoded session information to keep track of related client requests.
  • HTTP provides support for other functionalities such as cache-control (cache: Local storage/copy of a resource that is fetched from a server), content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining etc.

    • HTTP is invented by Sir Tim Berners-Lee at CERN in 1989. (CERN is a European organization)

    Jointly developed by W3C and IETF.

    HTTP Resource Addressing

    These are indentified by using URL's (Uniform Resource Locator). The schemes that are used in HTTP are "http" and "https". The default port for http (Hypertext Transfer Protocol) is 40 while in https (Hyper Text Transfer Protocol Secure), the default port is 443.

    The domain name is resolved to an IP address of the server where the resource resides or will be created.

    Query in HTTP

    The HTTP Query started of with a "?" sign. It is provided as a key = value pairs, with ampersand (&) separators between key/value pairs.

    It may also be a URL encoded, which means it resides on the client side. If it is URL decoded, it is in the server side.

    HTTP Request Message

    First, it goes to the request line. The order of HTTP request message are separated values, method, and request message. The message headers typically includes the general request/entity headers.

    In HTTP 1.1, it requires at least the host header. It also has an empty line and a message body or payload, which is optional in usage.
