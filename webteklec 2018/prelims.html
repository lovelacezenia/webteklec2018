    Hyper Text Transfer Protocol (HTTP) Fundamentals
  • HTTP typically runs on top of TCP/IP, using TCP port 80 by default (TCP port 443 for HTTPS).
  • HTTP resources are identified using URIs (specifically HTTP URLs).
  • HTTP is based on client-server architecture.
  • HTTP is not the most visible but the corner stone of how web works.
  • HTTP is an application layer communication protocol used to access resources (hypertext/hypermedia)on WWW.
  • HTTP is a pull protocol, it is always the client who initiates or pull communication.
  • Clients (AKA User Agents (UA)).
    - Web browsers, Web crawlers, Email clients, Other end user tools and applications
  • Servers
    - Origin servers, Proxy servers, Gateways, Tunnels
  • HTTP uses a request-response standard protocol.
    • - Client sends an HTTP request to the server.
    • - Server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol.
    • - Servers DO NOT keep information about clients between requests.
    • - Web applications affect session tracking using mechanisms such as cookies (a very small text file) on URL-encoded session information to keep track of related client requests.
  • HTTP provides support for other functionalities such as cache-control (cache: Local storage/copy of a resource that is fetched from a server), content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining etc.

    • HTTP is invented by Sir Tim Berners-Lee at CERN in 1989. (CERN is a European organization)

    Jointly developed by W3C and IETF

  • HTTP provides support for other functionalities, such as:
    • - cache control
    • - Content Media Type (MIME) Specification
    • - language and character set specification
    • - content/transfer coding
    • - content negotiation

    HTTP Resource Addressing

    These are indentified by using URL's (Uniform Resource Locator). The schemes that are used in HTTP are "http" and "https". The default port for http (Hypertext Transfer Protocol) is 40 while in https (Hyper Text Transfer Protocol Secure), the default port is 443.

    The domain name is resolved to an IP address of the server where the resource resides or will be created.

    Query in HTTP

    The HTTP Query started of with a "?" sign. It is provided as a key = value pairs, with ampersand (&) separators between key/value pairs.

    It may also be a URL encoded, which means it resides on the client side. If it is URL decoded, it is in the server side.

    HTTP Request Message

    First, it goes to the request line. The order of HTTP request message are separated values, method, and request message. The message headers typically includes the general request/entity headers.

    In HTTP 1.1, it requires at least the host header. It also has an empty line and a message body or payload, which is optional in usage.

    HTML

    The HTML (Hypertext Markup Language) is a language that is used to markup documents across the World Wide Web. It is used to specify the contents of webpages.

    HTML is solely for semantic specification of webpages. It is initally developed at CERN, then the IETF, then the W3C, and then the WHATWG.

    HTML Version History
  • Version History
    • - Tim Berners-Lee created the HTML Tags in the late 1991.
    • - The HTML Draft was created by Tim Berners-Lee & Dan Connolly and it was published by the IETF in 1993.
  • HTML 2.0
    • - The original documentation can be found by the RFC 1866, and it was created in November 1995.
    • - This version of HTML has the use of a Client Side Image Maps where it has an image with clickable areas.
    • - It has a form based upload that allows the producer to request information from the user.
    • - It has an HTML tables for any tabular information that is followed by one or more rows.
    • - It has also a Internationalization to give any additional recommendations for proper support.
  • HTML 3.2
    • - HTML 3.2 was the first version of HTML to be implemented and released as a W3C reccomendation and because in HTML 3.2, it was not propesed or published entirely.
    • - It was released on January 14th, 1997.
    • - It was the first set of HTML that is standardized and recognized by the W3C.
    • - The representatives that is involved in the standards of HTML 3.2 is from different companies like Netscape, Microsoft, Sun Microsystems, etc
  • HTML 4.0
    • - HTML 4.0 was published as a W3C reccomendation in December 18, 1997.
    • - It is initially named "Cougar".
    • - This version of HTML introduces CSS that has the ability to control multiple web pages at once.
    • - The improvement of tables have already been introduced in this version to provide additional information for Braille or speech users.
  • HTML 4.01
    • - HTML 4.01 was published as a W3C reccomendation in December 24, 1999.
    • - There was some errors in the specifications of HTML 4.01.
    • - This version of HTML is the accepted standard and the majority of web users are using this HTML Version.
    • - It has a Strict, Transitional, and Frameset, document type definition.
    XHTML Version History
  • XHTML
    • - XHTML is the reformulation of HTML.
    • - This was intended to introduce the new HTML elements and attributes as well. It's initial release is January 26, 2000.
    • - This version of HTML is the accepted standard and the majority of web users are using this kind of HTML Version.
    • - It has a Strict, Transitional, and Frameset, document type definition.
  • XHTML 1.0
    • - XHTML 1.0 was officially adopted by the W3C Recommendation in January 2000.
    • - XTHML 1.0 Strict includes elements that are not deprecated in HTML 4.01.
    • - XHTML 1.0 Transitional is equal to the HTML 4.01 Transitional, and includes the elements such as font, strike, and center.
    • - XHTML 1.0 Frameset allows the definition of some frameset documents such as the iframe element.
  • XHTML 1.1
    • - XHTML 1.1 becomes a W3C Recommendation in May 2001.
    • - XTHML 1.1 Strict includes elements that are not deprecated in HTML 4.01.
    • - XHTML 1.1 drops all derecated elements from XHTML 1.0.
    • - XHTML 1.1 is designed towards the styles or sheets of document structures.
